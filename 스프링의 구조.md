## controller
* 사용자의 요처을 받기
* 사용자의 요청 처리
* 요청 전달
## service
* repository - controller를 연결함(미들웨어)
* DB를에 데이터 전달 및 가공(입력,전달 둘다 가능)
## repository
* controller -> 요청-> repository(CRUD(DB명령어)작성) 
* 클라이언트의 요청을 받아 DB에 어떻게 할지를 작성함
## entity
* DB에 만들어진 속성들을 하나의 객체로 만든것임
    * DB에 접근하기 편함
## dto 
* controller(사용자)가 받고 요청한 값을 전달함
```java
@PostMapping("/users")
public void save(UserDTO dto)//매개변수(파라미터)가 DTO임 받는 값
 {
    userService.save(dto);
}
```
